HI，海斌，这是我在开发 IDSP 时，遇到的问题，并做的总结。大致如下：


（1）每个账户都有配额上的限制，配额消耗完后，API的所有操作都无法进行

（2）API 请求后，返回的数据，和我们设计的数据类型，有的不一致，需要进行转化或处理

（3）在进行推送数据时，推送的数据大多数有限制（数值、数据类型等），需要在推送时，进行验证、处理

（4）在拉取效果数据时，有的效果数据报告尚未生成，在入库前，需进行数据的验证

（5）在用 MCC账户拉取时，只能拉取到 【操作人】是 MCC 的数据，如果操作人是单账户的话，用MCC账户，拉取不到。在 从渠道下载的 操作历史数据 excel 表格中，操作人包括了 MCC 账户  和 单账户，所以数据不匹配，所以，在拉取 历史操作数据 时，需要 MCC 账户 和 单账户  同时拉取。

（6）有些历史操作数据，从API中，拉取过来的数据，本身就和平台中的相应数据不一致。

（7）如果，需要拉取搜狗历史的效果数据的话，需要将 idsp_channel_stat_day 中，相应日期的数据删除，再重新拉取

（8）SEM 渠道新增账户后，不仅需要在 CMS 中添加账户，还需要在 定时任务脚本中，添加拉取该账户的相关数据的定时脚本（拉取线上词库数据）

（9）如果 SEM 需要修改现有 CJ_CHANNEL 中推广账户的密码，需在推广平台 和 CMS 平台，同时修改该账户密码

（10）在 SEM 新增推广账户时，在 CMS 中添加该账户信息时，需要到推广账户的 API 页面中，得到该账户的 API token，添加到当前账户的相应 token 项中。
